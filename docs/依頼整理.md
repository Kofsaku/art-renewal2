## 3. 具体的なタスク内容

### タスク1：画面レイアウト再構築（内容②）

対象となる6画面のレイアウトを構築してください。

* 
**対象画面：** ①個人一覧、②個人登録/編集、③報告書、④データモニター、⑤ステータスモニター、⑥マップモニター 


* **共通要件：**
* Googleフォトのような「ハンバーガーメニュー」を採用 


* 表示領域の横幅は可変とし、最小・最大幅を設定すること 


* 1画面完成時点で中間レビューを行う（この時点では仮アイコンで可） 





### タスク2：画面遷移機能の実装（内容③）

以下の画面遷移およびポップアップ表示の実装をお願いします。

* 
**新規実装：** データモニターから個人登録画面への遷移 


* **ポップアップ統一（実装済み箇所の修正）：**
* 個人登録から報告書への遷移 


* ステータスモニターから報告書への遷移 


* 一斉操作および遠隔操作時のポップアップ 





### タスク3：アイコン考案と適用（内容①）

合計17種のアイコンを製作し、各画面のメニューやタブ、左上アイコンとして適用してください。

* 
**内訳：** メインメニュー用 12種、システムメニュー用 5種 


* **要件：**
* 白黒印刷時（モノトーン）でも各機能の識別ができること 


* 2〜3個完成した時点で中間レビューを実施 





### タスク4：カメラ連動デモ用ソフト製作（内容④）

特定の画面においてカメラ映像を連動させる機能をデモ用に製作してください。

* 
**対象：** 2画面分（データモニター等からの連携を想定） 


* 
**機能要件：** 警報発生時の録画映像表示、ライブ映像の視聴およびズーム機能 


* 
**備考：** API関連の資料は、タスク1〜3の完了後に支給されます

## 4. テクニカルメモ・補足事項

* 
**デザイン方針：** 時代変化による都度改修が不要な、シンプルで直感的なデザインを目指す 


* 
**ステータス表示：** ゲートの状態（開扉/閉扉、解錠/施錠、正常/警告/異常/オフライン）を視覚的に分かりやすく表示すること 


* 
**参考資料：** 既存ソフト「XSS」のアイコン機能一覧に基づき、必要な機能を網羅してください
